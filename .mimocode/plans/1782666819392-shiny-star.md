# Sistema de UI Modular com Sprite e Health Bar

## Contexto

O engine já tem:
- `Entity` com `health`, `max_health`, `take_damage()`, `heal()` (bug: checa `val < max_health` em vez de `health < max_health`)
- `UI` com `renderHUD()` que só mostra dinheiro
- `Texture` class que carrega PNG/JPG via stb_image → OpenGL texture
- ImGui já integrado via GLFW+OpenGL3
- Pasta `src/ui/ui_comps/` vazia, pronta para uso
- Nenhum uso de `ImGui::Image()` ou `ImTextureID` no código atual

**Objetivo:** Criar um sistema modular de UI components com sprite rendering e uma health bar que lê `Entity::health`.

## Arquivos a criar

### 1. `src/ui/ui_comps/ui_component.h` — Base abstrata
```cpp
struct UIComponent {
    virtual ~UIComponent() = default;
    virtual void render() = 0;
    bool visible = true;
};
```
Simples, leve. Cada componente de UI herda disso.

### 2. `src/ui/ui_comps/sprite.h` — Renderização de sprites
Classe utilitária (header-only) que encapsula:
- `ImTextureID` + UV coords para texturas
- Cores sólidas (modo fill)
- Métodos: `draw()`, `drawFilled()`, `drawProgressBar()`
- Conversão automática `Texture::id()` → `ImTextureID`

Sem dono de textura — referencia texturas existentes.

### 3. `src/ui/ui_comps/health_bar.h` + `health_bar.cpp` — Health Bar
- Referência pointer para `Entity` (não owns)
- Sprite de fundo (cor escura) + sprite de preenchimento (cor dinâmica)
- Largura/altura configuráveis
- Posição: canto inferior esquerdo (Anchor::BottomLeft, padding 20px)
- Cor muda: verde → amarelo → vermelho baseado no % de saúde
- Renderiza via `ImDrawList` (retângulos coloridos, sem textura necessária)
- `setEntity(Entity*)` para ligar/desligar a entidade alvo

## Arquivos a modificar

### 4. `src/entity/entity.h` — Bugfix no heal()
- Trocar `if (val < max_health)` por `health = glm::min(health + val, max_health)`

### 5. `src/ui/ui.h` — Adicionar sistema de components
- Adicionar `#include "ui_comps/ui_component.h"`
- Adicionar membro: `std::vector<std::unique_ptr<UIComponent>> m_components`
- Adicionar método: `template<typename T, typename... Args> T& addUIComponent(Args&&... args)`
- Adicionar método: `template<typename T> T* getUIComponent()`
- Chamar `render()` de cada componente visível no `renderHUD()`

### 6. `src/ui/ui.cpp` — Renderizar components no HUD
- No `renderHUD()`, iterar `m_components` e chamar `render()` em cada um visível

### 7. `src/app/app.cpp` — Criar e conectar a health bar
- Após criar o player, adicionar health bar via `m_ui->addUIComponent<HealthBar>()`
- Passar referência do player para a health bar
- Passar pointer da ui no init (já existe `m_ui.get()`)

## Estrutura final

```
src/ui/ui_comps/
├── ui_component.h      (base abstrata)
├── sprite.h            (utilitário de rendering)
├── health_bar.h        (declaração)
└── health_bar.cpp      (implementação)
```
